## 2026-06-23 - Bypass UTF-8 decoding overhead for ASCII validations
**Learning:** When evaluating string patterns that are expected to be pure ASCII (like hex strings), using `.chars().all(...)` or iterating over characters with `.chars()` incurs unnecessary UTF-8 boundary checking overhead. The workspace pattern for hot-path performance accepts substituting this with byte-level checks.
**Action:** When performing ASCII-only numeric formatting or string processing, avoid `.chars().all(...)` or `.chars().collect::<Vec<char>>()`. Instead, use `.as_bytes().iter().all(u8::is_ascii_hexdigit)` to check ASCII properties, or `.into_bytes()` to retrieve a `Vec<u8>` directly. Note: Use function pointers directly (like `u8::is_ascii_hexdigit`) to avoid `clippy::redundant_closure`.
