## 2026-06-20 - [Avoid UTF-8 checking in hot-path loops]
**Learning:** Operating directly on byte slices (`&[u8]`) instead of iterating through `String::chars()` or converting to `Vec<char>` allows bypassing of UTF-8 boundary check overhead when performing ASCII-only text processing and string substitutions (e.g. sanitizing characters, replacing parts of strings). This applies to string sanitizations on hot paths, where we can fall back to using `String::from_utf8(bytes).unwrap()` since we control the modified bytes and guarantee they are valid ASCII, yielding faster execution without sacrificing correctness.
**Action:** When validating or performing substitutions of pure ASCII strings, use `.as_bytes()` and iterate over bytes directly. Use `Vec<u8>` or stream processing, avoiding operations like `.chars().all(...)` or intermediate `Vec<char>`.
